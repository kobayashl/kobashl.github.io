Laravel環境構築(Homestead)

※Window環境

   |  小林

Homesteadを使用することで楽に開発環境の構築ができます。


事前準備

  • Gitのインストール
  • Vagrantのインストール
  • VirtualBoxのインストール

Homestead環境構築


Vagrant Boxのインストール
任意のフォルダで以下のコマンドを実行

vagrant box add laravel/homestead  #Vagrant Boxのインストール

以下のフォルダにlaravel-VAGRANTSLASH-homesteadが存在することを確認してください
デフォルトの場合C:/Users/username/.vagrant.d/boxes
※存在しない場合vagrantのバージョンを最新にしてください


Homesteadのチェックアウト
任意のフォルダで以下のコマンドを実行

git clone https://github.com/laravel/homestead.git /Users/username/Homestead #Homesteadのチェックアウト
cd /Users/username/Homestead #上記でチェックアウトしたHomesteadに移動

※homeディレクトリでなくても大丈夫ですが、ドキュメントに従いました
以下のコマンドを実行してHomestead boxのバージョンを確認します

vagrant box list #vagrant boxの一覧表示
laravel/homestead (virtualbox, 10.1.0)

出力結果の10.1.0にHomestead boxのバージョンを合わせます

git checkout v10.1.0
init.bat #Homestead.yamlの初期化

Homestead.yamlファイルの編集
Homestead/Homestead.yaml

folders: #共有フォルダの設定
	- map: C:/Users/username/code #ホスト側
      to: /home/vagrant/code #ゲスト側

Homestead boxの起動

vagrant up #Homestead boxの起動

laravelプロジェクトの作成

vagrant ssh #Homesteadにssh接続
cd code
composer create-project laravel/laravel projectName --prefer-dist #Laravelプロジェクトの作成

Homestead.yamlファイルの編集
Homestead/Homestead.yaml

sites: #公開フォルダの設定
    - map: homestead.test #ホスト側ドメイン名
      to: /home/vagrant/code/projectName/public #ゲスト側公開フォルダ

hostsファイルの編集
C:\Windows\System32\drivers\etc\hosts

192.168.10.10 homestead.test ドメイン名とIPアドレスの紐付け

サイトの確認

exit #vagrantから抜ける
vagrant reload --provision

http://homestead.test/にアクセスすると以下のようなディフォルトサイトが表示されます

以上、お疲れ様でした
